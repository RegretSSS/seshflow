# Seshflow Markdown 任务模板格式规范

## 目标

设计简洁的 Markdown 任务格式，让 AI 容易理解和生成，实现批量任务导入。

## 格式规则

### 基本语法

```markdown
## [阶段/分组名称]

- [ ] 任务标题 [优先级] [标签1,标签2] [预估工时]
- [ ] 另一个任务 [P1] [frontend,ui] [2h] [依赖:任务ID]
```

### 语法说明

1. **任务行格式**：
   ```
   - [ ] 任务标题 [P0] [tag1,tag2] [4h] [依赖:任务ID或序号]
   ```

2. **优先级**：
   - `[P0]` - 紧急
   - `[P1]` - 高
   - `[P2]` - 中（默认）
   - `[P3]` - 低

3. **标签**：
   - 用逗号分隔
   - 可选
   - 示例：`[auth,backend,database]`

4. **工时**：
   - 单位：小时（h）
   - 可选
   - 示例：`[4h]`, `[0.5h]`

5. **依赖关系**：
   - 可选
   - 引用其他任务
   - 格式：`[依赖:任务ID]` 或 `[依赖:序号]`

6. **分组/阶段**：
   - 使用 `##` 标题
   - 自动创建任务分组
   - 支持多级分组

### 完整示例

```markdown
# 项目初始化

## Phase 1: 基础设施

- [ ] 设计数据库架构 [P0] [database,design] [4h]
- [ ] 搭建开发环境 [P1] [devops] [2h]
- [ ] 配置 CI/CD [P2] [devops] [3h]

## Phase 2: 后端开发

- [ ] 实现 API 接口 [P0] [api,backend] [8h] [依赖:1]
- [ ] 实现认证系统 [P0] [auth,security] [6h] [依赖:4]
- [ ] 编写单元测试 [P1] [testing] [4h] [依赖:4,5]

## Phase 3: 前端开发

- [ ] 设计 UI 组件 [P1] [frontend,ui] [6h]
- [ ] 实现状态管理 [P2] [frontend] [4h]
- [ ] 集成 API [P1] [frontend,api] [5h] [依赖:4]

## Phase 4: 测试与部署

- [ ] 集成测试 [P1] [testing] [6h]
- [ ] 性能优化 [P2] [performance] [4h]
- [ ] 部署上线 [P0] [devops] [2h]
```

### 高级语法

#### 任务描述

在任务下方使用 `> quote` 添加描述：

```markdown
- [ ] 实现用户认证 [P0] [auth] [6h]
> 实现 JWT 认证系统，包括登录、注册、密码重置功能
> 技术栈：Node.js + Express + JWT
```

#### 子任务

使用嵌套列表：

```markdown
- [ ] 实现用户认证 [P0] [auth] [6h]
  - [x] 设计数据库表
  - [ ] 实现登录接口
  - [ ] 实现注册接口
  - [ ] 添加 JWT 验证
```

#### 分配人

```markdown
- [ ] 实现用户认证 [P0] [auth] [6h] [@张三]
```

## 解析规则

1. **任务识别**：
   - 以 `- [` 开头的行
   - 紧跟空格或 `x`（完成状态）
   - 然后是任务标题

2. **属性提取**：
   - 使用正则表达式匹配方括号内容
   - 优先级：匹配 `P0|P1|P2|P3`
   - 标签：逗号分隔的字符串
   - 工时：数字 + `h`
   - 依赖：`依赖:` 前缀

3. **分组结构**：
   - `##` 标题作为分组名称
   - 任务归属到最近的分组

4. **默认值**：
   - 优先级：`P2`
   - 工时：`0`
   - 标签：`[]`
   - 依赖：`[]`

## 使用示例

### 1. 创建任务文件

创建 `PROJECT_TASKS.md`：

```markdown
# Seshflow 项目开发

## CLI 核心功能

- [ ] 实现 init 命令 [P0] [cli] [2h]
- [ ] 实现 add 命令 [P0] [cli] [3h]
- [ ] 实现 next 命令 [P1] [cli] [2h]
- [ ] 实现 done 命令 [P1] [cli] [2h]

## Web 界面

- [ ] 创建 Web 包结构 [P0] [web,setup] [2h]
- [ ] 实现 Board 组件 [P0] [web,ui] [4h]
- [ ] 实现 Column 组件 [P1] [web,ui] [2h]
- [ ] 实现 Card 组件 [P1] [web,ui] [3h]
```

### 2. 导入任务

```bash
seshflow import PROJECT_TASKS.md
```

### 3. 查看导入的任务

```bash
seshflow next
```

## AI 友好特性

1. **简洁明确**：一行一个任务，易于生成
2. **结构清晰**：使用标准 Markdown 语法
3. **可读性强**：任务列表直观展示
4. **灵活扩展**：支持可选属性
5. **容错性好**：可选属性可省略

## 最小化示例

最简单的任务格式：

```markdown
# 项目任务

- [ ] 任务1
- [ ] 任务2
- [ ] 任务3
```

所有属性都是可选的！
