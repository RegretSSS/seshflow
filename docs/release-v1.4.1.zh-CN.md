# Seshflow v1.4.1

`v1.4.1` 是一个聚焦在高频任务流上的 hotfix 版本。

## 重点改动

- 新增 `seshflow done --start-next`，让 AI 或人工操作者可以在一条显式命令中完成当前任务并立刻启动下一项 ready task。
- 保持原有命令边界不变：
  - `next` 仍然是默认的顺序推进入口
  - `start <taskId>` 仍然是显式切换 / 显式指定任务入口
- README 和 CLI 文档已同步更新，说明什么时候该用快捷入口，什么时候继续用基础命令。

## 为什么需要这个版本

多轮真实测试反复暴露出这样一条常见链路：

1. `seshflow done`
2. `seshflow next`
3. `seshflow start <taskId>`

这条链路本身没有问题，但当操作者已经信任默认顺序推进时，会浪费一轮甚至两轮交互。`v1.4.1` 的目标就是在**不打乱命令模型**的前提下，提供一个更顺手的快捷入口。

## 边界没有变化

这个版本 **没有**：

- 把 `next` 和 `start` 合并成一个命令
- 把 `next` 改成纯选择器
- 改变 `start` 作为显式任务切换命令的定位

它只是增加了一个更窄、更克制的“完成后继续下一项”快捷入口。
