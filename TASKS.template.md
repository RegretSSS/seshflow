# 项目任务模板

> 使用此模板创建项目任务列表
> 复制此文件并重命名为 `PROJECT_TASKS.md`
> 然后运行 `seshflow import PROJECT_TASKS.md`

---

## 快速开始

### 最简单的任务列表

```markdown
# 我的任务

- [ ] 任务1
- [ ] 任务2
- [ ] 任务3
```

### 带属性的任务列表

```markdown
# 项目开发

## Phase 1: 基础功能

- [ ] 设计数据库 [P0] [database] [4h]
- [ ] 实现 API [P1] [api] [6h]
- [ ] 编写测试 [P2] [test] [3h]

## Phase 2: 前端开发

- [ ] 设计 UI [P1] [ui] [4h]
- [ ] 实现组件 [P1] [frontend] [6h]
```

---

## 完整语法示例

```markdown
# 项目名称

## 阶段/分组名称

- [ ] 任务标题 [优先级] [标签1,标签2] [预估工时] [@分配人] [依赖:任务ID]
  - [x] 已完成的子任务
  - [ ] 待完成的子任务
> 任务描述（可选）
> 可以是多行描述
```

---

## 优先级说明

- `[P0]` - 紧急（红色）
- `[P1]` - 高（橙色）
- `[P2]` - 中（黄色，默认）
- `[P3]` - 低（绿色）

---

## 标签示例

- `[auth]` - 认证相关
- `[database]` - 数据库相关
- `[api]` - API 相关
- `[frontend]` - 前端相关
- `[backend]` - 后端相关
- `[ui]` - UI/UX
- `[devops]` - 运维相关
- `[testing]` - 测试相关

---

## 工时说明

- `[4h]` - 4 小时
- `[0.5h]` - 30 分钟
- `[1d]` - 1 天（8 小时）

---

## 依赖关系示例

```markdown
- [ ] 设计数据库 [P0] [database] [4h]
- [ ] 实现 API [P1] [api] [6h] [依赖:1]
- [ ] 编写测试 [P2] [test] [3h] [依赖:2]
```

## 导入任务

创建好任务文件后，使用以下命令导入：

```bash
# 导入所有任务
seshflow import PROJECT_TASKS.md

# 查看下一个任务
seshflow next

# 查看所有任务
seshflow list
```

---

## 提示

1. **任务标题**：简洁明确，描述做什么
2. **优先级**：根据业务重要性设置
3. **标签**：便于分类和筛选
4. **工时**：合理预估，便于排期
5. **依赖**：明确任务间的关系
6. **子任务**：大任务分解为小任务
7. **描述**：提供足够上下文
